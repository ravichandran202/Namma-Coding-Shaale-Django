Python Dictionaries: Easy Notes

Think of a Dictionary like a real dictionary book. You look up a "word" (Key) to find its "meaning" (Value).

1. What is it?

  • It stores data in Key-Value pairs.
  • It is Mutable (you can change it later).
  • It does NOT allow duplicate Keys.
  • Note: In modern Python (3.7+), it remembers the order you added items.
Syntax Rule

Use curly braces {} and separate keys and values with a colon :

2. How to Create One

1. A Filled Dictionary 
student = {
    "name": "Ram",      # Key: "name", Value: "Ram"
    "age": 22,          # Key: "age",  Value: 22
    "course": "CS"      # Key: "course", Value: "CS"
}
2. An Empty Dictionary

You need this when you want to add data later.

# Correct way
empty_box = {}

# Also correct
empty_box = dict()

3. The Golden Rules

The KEY (The Label) The VALUE (The Content)
Must be Unique.
You can't have two keys named "age". 		Can be Duplicates.
Two students can have the same "age".
Must be Immutable.
Allowed: Strings, Numbers, Tuples.
NOT Allowed: Lists (because they change). 		Can be Anything.
Allowed: Strings, Lists, even other Dictionaries!

4. Changing Data (Mutability)

Dictionaries are mutable. You can update them anytime.

data = {"score": 10}

# 1. Update existing value
data["score"] = 20  
# Result: {"score": 20} -> The old 10 is GONE.

# 2. Add new pair
data["city"] = "Mysore"
# Result: {"score": 20, "city": "Mysore"}
Warning: Overwriting

If you assign a value to a Key that already exists, the old value is overwritten immediately.

5. The "Super Power" (Performance)

Why use a Dictionary instead of a List for big data?

Instant Access (O(1))

Imagine finding a book in a library.
List: You check every shelf one by one.
Dictionary: You use a computer to find the exact shelf instantly.
It takes the same amount of time (Constant Time) no matter how big the dictionary is.

The Secret: Hashing

Python uses a "Hash Table". It converts the Key into a unique number (hash) that points directly to where the value is stored in memory.
