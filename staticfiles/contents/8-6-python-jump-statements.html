Jump Statements (Break, Continue, Pass)

Jump statements are used to alter the normal flow of a loop. They allow you to terminate a loop prematurely, skip the current iteration, or define empty placeholders.

Keyword Action Common Analogy
break Terminates the loop entirely "Emergency Stop Button"
continue Skips current iteration, goes to next "Skip Song Button"
pass Does nothing (null operation) "To Be Done Later"

1. The 'break' Statement

The break statement is used to exit a loop immediately when a specific condition is met, regardless of whether the loop has finished iterating over all items.

# Program to find a number and stop searching
numbers = [10, 20, 30, 40, 50]
search_for = 30

for num in numbers:
    if num == search_for:
        print("Found it!")
        break  # Stops the loop here
    print(f"Checking {num}...")

print("Loop finished.") 
 Output
Checking 10...
Checking 20...
Found it!
Loop finished.
Observation
Notice that 40 and 50 were never checked. Once the break statement executed, the program "jumped" out of the loop completely.
Flow of 'break'
graph TD; A[Start Loop] --> B{Condition for break?}; B -->|Yes| C[Exit Loop Immediately]; B -->|No| D[Execute Code]; D --> A; C --> E[Continue rest of program];

2. The 'continue' Statement

The continue statement skips the rest of the code inside the current loop iteration and jumps back to the top of the loop for the next item.

# Program to print only odd numbers
for i in range(1, 6):
    if i % 2 == 0:     # If number is even
        continue       # Skip printing, go to next number
    print(i) 
 Output
1
3
5
Step-by-Step
  • i = 1: Odd. Prints 1.
  • i = 2: Even. continue triggers. Skips print.
  • i = 3: Odd. Prints 3.
  • i = 4: Even. continue triggers. Skips print.
Flow of 'continue'
graph TD; A[Start Iteration] --> B{Condition for continue?}; B -->|Yes| A; B -->|No| D[Execute Code]; D --> A;
The 'continue' statement creates a shortcut back to the start of the loop

3. The 'pass' Statement

The pass statement is a null operation. Nothing happens when it executes. It is useful as a placeholder when syntax requires a statement but you have no code to write yet.

# Using pass for empty functions or loops
for i in range(5):
    pass  # TODO: Write logic here later

def my_future_function():
    pass  # Prevents IndentationError

print("Code runs without errors!")

Common Mistakes with Jump Statements

  1. 1. Unreachable Code

    Placing code immediately after break or continue in the same block. This code will never run.

    # Incorrect: Code after continue is dead
for i in range(3):
    if i == 1:
        continue
        print("This will never print") # Warning: Unreachable code
    print(i)
    Logic Error
    Any lines indented directly under a break or continue statement will never be executed. Move them before the jump statement if they need to run.
  2. 2. Using 'break' outside a loop

    You cannot use break or continue inside a standalone if statement that isn't inside a loop.

    x = 10
if x > 5:
    break  # SyntaxError: 'break' outside loop
Practice Exercise

Scenario: You are processing a list of order statuses.
1. If the status is "Cancelled", skip it.
2. If the status is "Error", stop the entire process immediately.
3. Otherwise, print "Processing...".

Fix the bugs in the code below:

# Buggy code
orders = ["Confirmed", "Cancelled", "Confirmed", "Error", "Pending"]

for status in orders:
    if status == "Cancelled":
        break          # BUG 1: Should this stop everything?
    elif status == "Error":
        continue       # BUG 2: Should this just skip?
    
    print(f"Processing order: {status}")
# Corrected code
orders = ["Confirmed", "Cancelled", "Confirmed", "Error", "Pending"]

for status in orders:
    if status == "Cancelled":
        continue       # Fix 1: Use continue to skip ONLY this item
    elif status == "Error":
        print("Critical Error found! Stopping.")
        break          # Fix 2: Use break to stop the loop entirely
    
    print(f"Processing order: {status}") 
 Output
Processing order: Confirmed
Processing order: Confirmed
Critical Error found! Stopping.
Correction Analysis
  • "Cancelled": We used continue because we want to ignore this one bad order but keep processing the rest.
  • "Error": We used break because a system error implies we should stop processing everything immediately to prevent issues.
