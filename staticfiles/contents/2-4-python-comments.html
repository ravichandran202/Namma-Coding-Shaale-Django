Python Comments

Comments are non-executable lines in your code. They are used to explain what the code is doing, making it easier for humans to read and maintain.

Why use comments?
  • Readability: Explains complex logic to teammates (or your future self).
  • Debugging: Temporarily prevent code execution without deleting it.
  • Documentation: Provides metadata about the file, author, or function.

1. Single-Line Comments

Python uses the hash symbol (#) for single-line comments. Anything after the # on that line is ignored by Python.

# This is a complete line comment
print("Hello World") # This is an inline comment

2. Multi-Line Comments

Python doesn't have a specific syntax for multi-line comments. You can either use multiple # symbols or Triple Quotes (multiline strings).

# This is a comment
# written in
# more than just one line
"""
This is a multiline string
used as a comment. Python ignores
it if not assigned to a variable.
"""
print("Multi-line demo")
