Python Variable Scope: Global and Local

Understand how variables work inside and outside functions with Python's scope rules.

What is Scope?

Scope determines where in your program a variable can be accessed and modified.

Global Variable: Declared outside all functions - accessible everywhere

Local Variable: Declared inside a function - accessible only within that function

Global Variable

A variable declared outside all functions. It can be accessed inside and outside functions.

# Global variable declared outside all functions
x = 20  # global

def my_func():
    print("Inside function:", x)  # Can access global variable

my_func()
print("Outside function:", x) 
 Output
Inside function: 20
Outside function: 20

1. Local Shadows Global

If a variable inside a function has the same name as a global variable, the local one is used inside the function.

x = 30  # global variable

def func():
    x = 100  # local variable shadows global 
    print("Inside function:", x)

func()
print("Outside function:", x)  # global stays unchanged 
 Output
Inside function: 100
Outside function: 30
What's Happening?

Inside func(), Python creates a new local variable x that temporarily "hides" the global x. The global variable remains unchanged.

2. Modifying Global Variables with global

The Problem

Without global keyword, you cannot modify global variables inside functions:

x = 50  # global

def my_func():
    # This would cause ERROR!
    # x = x + 10  # UnboundLocalError
    print("Cannot modify global without 'global' keyword")

my_func()
Solution: Use global keyword
x = 50  # global variable

def my_func():
    global x  # Tell Python we want to use the global x
    x = x + 10  # Now we can modify it
    print("Inside function:", x)

print("Before function:", x)
my_func()
print("After function:", x) 
 Output
Before function: 50
Inside function: 60
After function: 60
Remember

Use global keyword when you want to modify a global variable inside a function.

You can read global variables without global, but not modify them.

3. Creating Global Variables Inside Functions

def my_func():
    global y  # Declare y as global
    y = 200   # Create new global variable
    print("Inside function:", y)

# y doesn't exist here yet
my_func()     # This creates the global variable y
print("Outside function:", y)  # Now y exists globally 
 Output
Inside function: 200
Outside function: 200
Caution

Creating global variables inside functions is generally not recommended as it can make code harder to understand and debug.

Scope Search Rule: LEGB

LEGB Rule

When Python looks for a variable, it checks in this order:

  1. Local - Inside current function
  2. Enclosing - In outer functions (if nested)
  3. Global - defined outside functions
  4. Built-in - Python reserved names like len, print
LEGB Rule in Action
# Built-in: len, print, etc.
# Global: module-level variables
global_var = "I'm global"

def outer_function():
    # Enclosing: for inner_function
    enclosing_var = "I'm enclosing"
    
    def inner_function():
        # Local: only in this function
        local_var = "I'm local"
        print(local_var)        # Local - Found
        print(enclosing_var)    # Enclosing - Found  
        print(global_var)       # Global - Found
        print(len("hello"))     # Built-in - Found
    
    inner_function()

outer_function() 
 Output
I'm local
I'm enclosing
I'm global

Practical Examples

Example 1: Counter with Global Variable
# Global counter
count = 0

def increment_counter():
    global count
    count += 1
    print(f"Counter: {count}")

def reset_counter():
    global count
    count = 0
    print("Counter reset!")

# Test the counter
increment_counter()
increment_counter()
increment_counter()
reset_counter()
increment_counter() 
 Output
Counter: 1
Counter: 2
Counter: 3
Counter reset!
Counter: 1

Try It Yourself

Given the following code with global variables:

balance = 1000
username = "guest"

Create functions to:

  1. Deposit money (add to balance)
  2. Withdraw money (subtract from balance)
  3. Change username
  4. Display current account info

Test your functions with various operations.

# Global variables
balance = 1000
username = "guest"

def deposit(amount):
    global balance
    balance += amount
    print(f"Deposited ${amount}. New balance: ${balance}")

def withdraw(amount):
    global balance
    if amount <= balance:
        balance -= amount
        print(f"Withdrew ${amount}. New balance: ${balance}")
    else:
        print("Insufficient funds!")

def change_username(new_name):
    global username
    username = new_name
    print(f"Username changed to: {username}")

def display_info():
    print(f"User: {username}, Balance: ${balance}")

# Test the functions
display_info()
deposit(500)
withdraw(200)
change_username("alice123")
withdraw(2000)  # Should show insufficient funds
display_info() 
 Output

Withdrew $200. New balance: $1300
Username changed to: alice123
Insufficient funds!
User: alice123, Balance: $1300

Key Takeaways
  • Global variables are accessible everywhere
  • Local variables exist only within their function
  • Local variables with same name shadow global ones
  • Use global keyword to modify global variables inside functions
  • Python follows LEGB rule when searching for variables
  • Avoid creating globals inside functions - it makes code harder to debug
  • Use global variables for constants or configuration, not for frequent changes
