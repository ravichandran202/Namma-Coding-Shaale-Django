Lists vs Arrays

Both store multiple items, but have different purposes.

Feature Python List Python Array
What it is General-purpose container (like a backpack) Specialized tool for numbers (like an egg carton)
Data Types Can hold mixed types (numbers, text, etc.) Must hold only one type of number
Import Needed No - built into Python Yes - import array
Memory & Speed Uses more memory, slower for math Uses less memory, faster for math
Flexibility Very flexible - easy to add/remove items Less flexible - slower to change
When to Use Everyday use, mixed data types Large number sequences, better performance
Python List Examples
# Lists can hold mixed data types
my_list = [1, "hello", 3.14, True]
print("Mixed list:", my_list)

# Easy to add and remove
my_list.append("new item")
my_list.remove(3.14)
print("After changes:", my_list)

# Many built-in methods
numbers = [3, 1, 4, 1, 5]
numbers.sort()
print("Sorted:", numbers)
print("Count of 1:", numbers.count(1)) 
 Output
Mixed list: [1, 'hello', 3.14, True]
After changes: [1, 'hello', True, 'new item']
Sorted: [1, 1, 3, 4, 5]
Count of 1: 2
Python Array Examples
# Must import array module
import array

# Create array of integers
int_array = array.array('i', [1, 2, 3, 4, 5])
print("Integer array:", int_array)

# Create array of floats
float_array = array.array('f', [1.1, 2.2, 3.3])
print("Float array:", float_array)

# All elements must be same type
# This would cause error:
# mixed_array = array.array('i', [1, "hello"])  # TypeError! 
 Output
Integer array: array('i', [1, 2, 3, 4, 5])
Float array: array('f', [1.1, 2.2, 3.3])
Array Type Codes
Code Type Example
'i' Integer [1, 2, 3]
'f' Float [1.1, 2.2]
'd' Double [1.234, 5.678]
When to Use Which?
Use Lists When:
  • Storing different data types together
  • Frequently adding/removing items
  • Everyday programming tasks
  • Small to medium datasets
  • Need built-in methods like .sort()
Use Arrays When:
  • Working with large number sequences
  • Need better performance for math operations
  • Memory efficiency is important
  • All elements are the same numeric type
  • Interfacing with C/C++ code
