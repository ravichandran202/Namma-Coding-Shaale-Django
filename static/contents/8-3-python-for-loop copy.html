For Loops in Python

For loops allow us to iterate over a sequence (such as a list, tuple, string, or range) and execute a block of code for each item in the sequence.

For Loop Structure 
# Syntax
for item in sequence:
    # code to execute for each item
Basic For Loop Examples
# Program to print each character in a string
name = "Alex"  
for char in name:  
    print(char) 
 Output
A
l
e
x
Step-by-Step Execution
  1. Sequence: "Alex" (4 characters)
  2. Iteration 1: char = "A" → Print "A"
  3. Iteration 2: char = "l" → Print "l"
  4. Iteration 3: char = "e" → Print "e"
  5. Iteration 4: char = "x" → Print "x"
  6. No more items → Exit loop
# Program to print numbers from 0 to 4
for i in range(5):
    print(i) 
 Output
0
1
2
3
4
How range() Works
  • range(5) generates: 0, 1, 2, 3, 4
  • range(start, stop) generates from start to stop-1
  • range(start, stop, step) generates with step size
  • Stop value is exclusive (not included)

Common For Loop Mistakes

  1. 1. Missing Colon

    Forgetting the colon (:) at the end of the for statement.

    # Incorrect: Missing colon
for i in range(5)  # SyntaxError
    print(i) 
 Output
SyntaxError: invalid syntax
    # Correct: With colon
for i in range(5):  # Colon present
    print(i) 
 Output
0
1
2
3
4
    Error Analysis
    • Colon (:) indicates the start of a code block
    • Required after for, while, if, etc.
    • Missing colon causes immediate SyntaxError
  2. 2. Incorrect Indentation

    Forgetting to indent the code block inside the for loop.

    # Incorrect: Missing indentation
for i in range(3):
print(i)  # This will cause an IndentationError 
 Output
IndentationError: expected an indented block
    # Correct: Proper indentation
for i in range(3):
    print(i)  # 4 spaces indentation 
 Output
0
1
2
    Error Analysis
    • Python uses indentation to define code blocks
    • All statements inside loop must be indented
    • Standard practice: 4 spaces per indentation level
    • Inconsistent indentation causes IndentationError
  3. 3. Off-by-One Errors with range()

    Confusion about inclusive/exclusive boundaries in range().

    # Incorrect: Expecting 1 to 5 but getting 0 to 4
for i in range(5):
    print(i)  # Prints 0, 1, 2, 3, 4 
 Output
0
1
2
3
4
    # Correct: If you want 1 to 5
for i in range(1, 6):
    print(i)  # Prints 1, 2, 3, 4, 5 
 Output
1
2
3
4
5
    Range Function Behavior
    • range(stop): 0 to stop-1
    • range(start, stop): start to stop-1
    • range(start, stop, step): start to stop-1 with step
    • Stop value is always exclusive
    # More range() examples
print("range(3):", list(range(3)))
print("range(1, 4):", list(range(1, 4)))
print("range(0, 10, 2):", list(range(0, 10, 2)))
print("range(5, 0, -1):", list(range(5, 0, -1))) 
 Output
range(3): [0, 1, 2]
range(1, 4): [1, 2, 3]
range(0, 10, 2): [0, 2, 4, 6, 8]
range(5, 0, -1): [5, 4, 3, 2, 1]
  4. 4. Iterate over Numerical Value

    Trying to iterate over a single number instead of a sequence.

    # Incorrect: Trying to iterate over a single number
for i in 5:  # TypeError: 'int' object is not iterable
    print(i) 
 Output
TypeError: 'int' object is not iterable
    # Correct: Use range() for numerical loops
for i in range(5):
    print(i) 
 Output
0
1
2
3
4
    Error Analysis
    • For loops require iterable objects
    • Single numbers (int, float) are not iterable
    • Use range() to create sequences for numerical loops
    • Other iterables: lists, strings, tuples, dictionaries
Extra Tips & Best Practices
# Using _ for unused loop variables
# When index is not used
for i in range(5):
    print("Hello")  # i is not used

# Better: Use _ to indicate unused variable
for _ in range(5):
    print("Hello")  # Clearly shows we don't need the loop variable 
 Output
Hello
Hello
Hello
Hello
Hello
Hello
Hello
Hello
Hello
Hello
Why Use Underscore?
  • Clear intent: Shows the loop variable is intentionally unused
  • Code readability: Makes purpose obvious to other developers
  • Prevents warnings: Some IDEs/linters warn about unused variables
  • Python convention: Widely adopted in Python community
