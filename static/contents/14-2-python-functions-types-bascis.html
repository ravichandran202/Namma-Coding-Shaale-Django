Function Types

Learn about the four main types of functions based on parameters and return values

1. No Parameter, No Return

Functions that don't take input and don't return any value

def say_hello():
    print("Hello! Welcome to Python!")

# Calling the function
say_hello()
say_hello()
 Output
Hello! Welcome to Python!
Hello! Welcome to Python!

2. Parameter, No Return

Functions that take input but don't return any value

def greet(name):
    print(f"Hello {name}!")

# Calling with different arguments
greet("Alice")
greet("Bob")
greet("Namma Coding Shaale")
 Output
Hello Alice!
Hello Bob!
Hello Namma Coding Shaale!

3. No Parameter, With Return

Functions that don't take input but return a value

def get_greeting():
    return "Good morning!"

# Using the returned value
message = get_greeting()
print(message)
 Output
Good morning!

4. Parameter, With Return

Functions that take input and return a value

def add_numbers(a, b):
    result = a + b
    return result

# Using the returned value
sum = add_numbers(5, 3)
print("5 + 3 =", sum)

# Using in expressions
total = add_numbers(10, 20) + add_numbers(5, 5)
print("Total:", total)
 Output
5 + 3 = 8
Total: 40

Common Errors

5. The Mystery of 'None'

What happens if you try to print or store the result of a function that has no return statement?

Important Concept
If a function does not have a return keyword, Python automatically returns a special value called None. It represents "nothing" or "empty". 
def say_hi(name):
    # This function prints, but DOES NOT return
    print(f"Hi {name}")

# This looks correct:
say_hi("Raju")

# But look what happens here:
result = say_hi("Raju") 
print("The stored value is:", result)
 Output
Hi Raju
The stored value is: None

6. Common Error: Argument Mismatch

You must provide exactly the number of inputs the function expects.

If a function defines 2 parameters, you cannot send 1, and you cannot send 3. You must send exactly 2.

def subtract(a, b):
    print(a - b)

# Case 1: Too few arguments (Missing one)
subtract(10)  

# Case 2: Too many arguments (Extra one)
subtract(10, 5, 2)
 Output (Errors)
# Error for Case 1:
TypeError: subtract() missing 1 required positional argument: 'b'

# Error for Case 2:
TypeError: subtract() takes 2 positional arguments but 3 were given
Debugging Tip
When you see a TypeError related to arguments, count how many variables are inside the function definition def name(...) and make sure your function call name(...) has the same amount.

Quick Reference
When to use each type:
  • No Param, No Return: Simple messages, menus
  • Param, No Return: Display personalized info
  • No Param, With Return: Get fixed values
  • Param, With Return: Calculations, data processing
Remember:
  • Use print() for displaying
  • Use return for giving back values
  • Parameters make functions flexible
  • Return values make functions reusable

Choose the right type based on what your function needs to do!
