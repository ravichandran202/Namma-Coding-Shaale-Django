Break and Continue

Control flow statements alter the normal flow of execution. Two key statements, break and continue, allow us to precisely control loops in Python.

1. The Break Statement

What is Break?

The break statement terminates the loop containing it. Control of the program flows immediately to the statement after the body of the loop.

Real World Analogy

Imagine you are looking for a specific book in a stack. Once you find it, you stop looking (break) and walk away. You don't check the remaining books.

# Program to find a specific number
numbers = [1, 3, 5, 7, 8, 9, 11]

for num in numbers:
    if num == 8:
        print("Found 8! Stopping the loop.")
        break  # Exit the loop immediately
    print(f"Checking {num}...")

print("Loop finished.") 
 Output
Checking 1...
Checking 3...
Checking 5...
Checking 7...
Found 8! Stopping the loop.
Loop finished.
Break Flow Logic
flowchart TD A["Start Loop"] --> B{"Items left?"} B -- No --> E["End Loop"] B -- Yes --> C{"Is Condition Met?"} C -- "Yes (Break)" --> E C -- No --> D["Execute Loop Body"] D --> B
When the break condition is met, the loop terminates immediately.

2. The Continue Statement

What is Continue?

The continue statement skips the rest of the code inside the loop for the current iteration only. The loop does not terminate but continues with the next iteration.

Real World Analogy

Imagine listening to a playlist. If a song comes on that you don't like, you hit "Skip" (continue). The playlist doesn't stop; it just moves to the next song.

# Program to print only odd numbers
for i in range(1, 6):
    if i % 2 == 0:
        print(f"Skipping even number: {i}")
        continue  # Skip the rest of this iteration
    print(f"Printing odd number: {i}") 
 Output
Printing odd number: 1
Skipping even number: 2
Printing odd number: 3
Skipping even number: 4
Printing odd number: 5
Continue Flow Logic
flowchart TD A["Start Loop"] --> B{"Items left?"} B -- No --> E["End Loop"] B -- Yes --> C{"Is Condition Met?"} C -- "Yes (Continue)" --> B C -- No --> D["Execute Loop Body"] D --> B
When continue runs, it jumps back to the start (next item) immediately.

Break vs Continue

Feature Break Continue
Action Terminates the loop completely. Terminates the current iteration only.
Next Step Executes code outside the loop. Executes the next iteration of the loop.
Analogy Emergency Stop Button. Skip/Next Button.

Common Mistakes

  1. 1. The Infinite While Loop

    Using continue in a while loop without updating the counter often leads to infinite loops.

    # DANGER: Infinite Loop
i = 0
while i < 5:
    if i == 3:
        continue  # i never increments! Stuck at 3 forever.
    print(i)
    i += 1
    How to fix
    Ensure you increment your control variable before the continue statement inside a while loop.
  2. 2. Unreachable Code

    Any code placed immediately after break or continue inside the same block will never run.

    for i in range(3):
    break
    print("This will never print") # Dead code
Practice Challenge

Scenario: You are processing a list of product prices.

  • The list: prices = [10, 50, 0, 200, 150, 0, 30]
  • If the price is 0, it's invalid data: skip it.
  • If the price is above 100, it's too expensive: stop processing completely.
  • Print all valid, affordable prices.
Write the loop.

prices = [10, 50, 0, 200, 150, 0, 30]

for price in prices:
    if price == 0:
        continue  # Skip invalid data
        
    if price > 100:
        print("Price too high! Stopping.")
        break  # Stop processing
        
    print(f"Processing price: ${price}") 
 Output
Processing price: $10
Processing price: $50
Price too high! Stopping.
