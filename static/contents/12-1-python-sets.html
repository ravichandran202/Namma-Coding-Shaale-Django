Python Sets

Sets are unordered collections of unique, immutable elements - perfect for removing duplicates and membership testing.

  • Unordered: Items have no defined order
  • Mutable: You can add or remove items after creation
  • No Duplicates: Cannot have items with the same value
  • Immutable Elements: All elements must be hashable (numbers, strings, tuples)
  • Heterogeneous: Can contain elements of different data types
Syntax
# Create a set
set_name = {item1, item2, ...}

# Empty set (note: {} creates a dict!)
empty_set = set()
Example
my_set = {10, "Hello", 3.5, True}
print(my_set)
 Output
{True, 3.5, 10, 'Hello'}

1. Creating Sets

Create sets using curly braces or the set() constructor

# Set of numbers
numbers = {1, 2, 3, 4, 5}
print("Numbers:", numbers)

# Set of strings
fruits = {"apple", "banana", "cherry"}
print("Fruits:", fruits)

# Set with mixed data types
mixed = {1, "hello", 3.14, True}
print("Mixed:", mixed)

# Empty set (important!)
empty_set = set()
print("Empty set:", empty_set)

# From a list (removes duplicates)
duplicate_list = [1, 2, 2, 3, 3, 3]
unique_set = set(duplicate_list)
print("From list:", unique_set)
 Output
Numbers: {1, 2, 3, 4, 5}
Fruits: {'cherry', 'banana', 'apple'}
Mixed: {1, 3.14, 'hello'}
Empty set: set()
From list: {1, 2, 3}
Important Notes
  • {} creates an empty dictionary, not a set
  • Use set() to create an empty set
  • Sets automatically remove duplicate values
  • Elements must be immutable (cannot have lists or other sets as elements)

2. Set Characteristics

Unordered Nature

Sets don’t maintain order — the elements may appear in any sequence.
Even if two sets have the same elements in different orders, they’re considered equal.

# Sets don't maintain order
set1 = {"a", "b", "c"}
set2 = {"c", "b", "a"}

print("Set1:", set1)
print("Set2:", set2)
print("Are they equal?", set1 == set2)
 Output
Set1: {'c', 'a', 'b'}
Set2: {'a', 'c', 'b'}
Are they equal? True
No Duplicates

Sets automatically remove duplicate values. If you try to add the same element again, it will be ignored.

# Duplicates are automatically removed
numbers = {1, 2, 2, 3, 3, 3, 4, 4, 4, 4}
print("After adding duplicates:", numbers)

# Even if you try to add duplicates
colors = {"red", "green", "blue"}
colors.add("red")  # This won't add anything
print("After adding duplicate 'red':", colors)
 Output
After adding duplicates: {1, 2, 3, 4}
After adding duplicate 'red': {'blue', 'green', 'red'}
Immutable Elements Only

Only immutable (unchangeable) objects can be stored inside a set. You can add numbers, strings, or tuples — but not lists, dicts, or other sets.

# Valid: numbers, strings, tuples
valid_set = {1, "hello", (1, 2, 3)}
print("Valid set:", valid_set)

# Invalid: lists, dictionaries, other sets
# invalid_set = {1, [2, 3]}  # This would cause TypeError
# invalid_set = {1, {2, 3}}  # This would cause TypeError
 Output
Valid set: {1, 'hello', (1, 2, 3)}
Set Characteristics Summary
Feature Description Example
Unordered No defined order of elements {1, 2, 3} could display as {3, 1, 2}
Unique Elements Automatically removes duplicates {1, 2, 2, 3} becomes {1, 2, 3}
Fast Membership Very fast in operations item in set is O(1)

Use sets when: You need to store unique elements, remove duplicates, or perform fast membership testing.
