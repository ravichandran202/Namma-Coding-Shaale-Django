How Python Works

reference - medium.com

Compiler

The Python compiler is the component that transforms your human-readable Python source code (.py files) into bytecode that the Python Virtual Machine (PVM) can execute. It's part of the Python interpreter.

The popular Python compiler is CPython, which is written in C.

Bytecode

Bytecode is a low-level representation of Python source code. It is a set of instructions that the Python Virtual Machine ( PVM ) can execute.
Python Compiler generates bytecode when it compiles Python source code.

Python Virtual Machine

An engine that executes Python bytecode. It is the final layer in Python's execution process, running the compiled bytecode on your system.

How Python Code Runs

Python converts your code into bytecode, then the Python Virtual Machine (PVM) executes it line by line and shows the output.

Let's understand with example

  1. You Write the Code

    • Example: print("Hello") (like writing instructions in English)

  2. Python’s Secret Helper (Compiler)

    • Quickly changes your code into a simpler version called bytecode (.pyc file).

    • Like translating English into a secret code that’s easier for the next helper to read.

  3. Python’s Main Helper (Interpreter)

    • Reads the bytecode line by line.

    • Turns each line into 0s and 1s (computer language) and makes the computer do it.

Why is Python called interpreted?

  • The Interpreter does the real work while the program runs, step by step.
  • The Compiler just helps prepare the code faster.

example

  • The compiler quickly gets the recipe ready (turns it into bytecode).
  • The interpreter cooks the meal, step-by-step (takes the bytecode and turns it into actions the computer can do).

Summary

  • The Python Compiler turns your code into bytecode.
  • Bytecode is a platform-independent, low-level form of your program.
  • The Python Virtual Machine (PVM) executes bytecode line by line.
  • Python is called interpreted because the PVM runs code step by step at runtime.
  • Think of it like cooking: compiler = prepares recipe, interpreter = cooks step by step.
