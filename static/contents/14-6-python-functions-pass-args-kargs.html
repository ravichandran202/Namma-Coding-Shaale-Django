Python Functions: pass, *args, and **kwargs

Learn how to create flexible functions using pass, *args, and **kwargs for powerful Python programming.

1. pass Statement

A placeholder that does nothing. Used when syntax requires code but you don't want to write any.

# Empty function using pass
def my_function():
    pass  # Does nothing, but prevents error

my_function()  # No output, but no error either
When to Use pass
  • Creating empty functions/classes
  • Placeholder for future implementation
  • When you need syntactically correct but empty code

2. *args (Arbitrary Arguments)

Allows a function to accept any number of positional arguments.

Without *args (Limited)
def add_two_numbers(a, b):
    return a + b

print(add_two_numbers(5, 3)) 
 Output
8
Problem

add_two_numbers(5, 3, 2) would cause ERROR! ❌

With *args (Flexible)
def add_numbers(*args):
    print(f"Args received: {args}")
    return sum(args)

print(add_numbers(1, 2)) 
 Output
Args received: (1, 2)
3
print(add_numbers(1, 2, 3, 4, 5)) 
 Output
Args received: (1, 2, 3, 4, 5)
15
print(add_numbers()) 
 Output
Args received: ()
0
More *args Examples
# Example 2: Mixed parameters
def greet(message, *names):
    print(f"{message} to:")
    for name in names:
        print(f"- {name}")

greet("Welcome", "Alice", "Bob", "Charlie") 
 Output
Welcome to:
- Alice
- Bob
- Charlie
# Example 3: Find average
def calculate_average(*numbers):
    if len(numbers) == 0:
        return 0
    return sum(numbers) / len(numbers)

print(f"Average: {calculate_average(10, 20, 30, 40)}")
print(f"Average: {calculate_average(5, 15)}") 
 Output
Average: 25.0
Average: 10.0
Key Points about *args
  • *args collects extra positional arguments as a tuple
  • The name 'args' is convention - you can use any name after *
  • *args must come after regular parameters
  • You can access args like a normal tuple

3. **kwargs (Arbitrary Keyword Arguments)

Allows a function to accept any number of keyword arguments.

Without **kwargs (Limited)
def greet(name, age, city):
    print(f"Name = {name}")
    print(f"Age = {age}")
    print(f"City = {city}")

greet(name="Alice", city="New York", age=25) 
 Output
Name = Alice
Age = 25
City = New York
Problem

greet(name="Alice", age=25, city="NY", country="USA") would cause ERROR! ❌

With **kwargs (Flexible)
def greet(**kwargs):
    for key, value in kwargs.items():
        print(f"{key} = {value}")

greet(name="Alice", age=25, city="New York", gender="Male") 
 Output
name = Alice
age = 25
city = New York
gender = Male
More **kwargs Examples
# Example 2: Create user profile
def create_user_profile(**user_info):
    print("User Profile Created:")
    print("-" * 20)
    for field, value in user_info.items():
        print(f"{field.title()}: {value}")

create_user_profile(
    username="alice123",
    email="alice@email.com",
    age=25,
    profession="Engineer",
    city="Bangalore"
) 
 Output
User Profile Created:
--------------------
Username: alice123
Email: alice@email.com
Age: 25
Profession: Engineer
City: Bangalore
Key Points about **kwargs
  • **kwargs collects extra keyword arguments as a dictionary
  • The name 'kwargs' is convention - you can use any name after **
  • **kwargs must come after *args and regular parameters
  • You can access kwargs like a normal dictionary
Quick Reference
Feature Purpose Syntax Data Type
pass Empty placeholder pass -
*args Unlimited positional arguments def func(*args) Tuple
**kwargs Unlimited keyword arguments def func(**kwargs) Dictionary

Try It Yourself

Create a function called student_report that:

  1. Takes a required parameter for student name
  2. Accepts any number of subject scores using *args
  3. Accepts any additional information using **kwargs
  4. Calculates and displays the average score
  5. Shows all additional information

Test your function with:

student_report("Alice", 85, 92, 78, grade="A", attendance="95%")
def student_report(name, *scores, **details):
    print(f"Student: {name}")
    
    # Calculate average score
    if scores:
        average = sum(scores) / len(scores)
        print(f"Scores: {scores}")
        print(f"Average: {average:.1f}")
    else:
        print("No scores provided")
    
    # Display additional details
    if details:
        print("Additional Details:")
        for key, value in details.items():
            print(f"  {key}: {value}")

# Test the function
student_report("Alice", 85, 92, 78, grade="A", attendance="95%") 
 Output
Student: Alice
Scores: (85, 92, 78)
Average: 85.0
Additional Details:
  grade: A
  attendance: 95%

Test Your Knowledge

1. What does *args collect in a function?
Correct! *args collects extra positional arguments as a tuple.
Incorrect. *args handles positional arguments, not keyword arguments.
2. When should you use the pass statement?
Correct! pass is used as a placeholder when you need syntactically correct but empty code.
Incorrect. pass doesn't stop execution or transfer data.
3. What is the correct order of parameters in a function?
Correct! The order must be: regular parameters, *args, then **kwargs.
Incorrect. Python has strict rules about parameter order.
Key Takeaways
  • pass: Use as a placeholder for empty code blocks
  • *args: Collects unlimited positional arguments as a tuple
  • **kwargs: Collects unlimited keyword arguments as a dictionary
  • Order matters: regular → *args → **kwargs
  • These features make functions extremely flexible and reusable
  • Great for creating APIs, configuration functions, and utility functions
