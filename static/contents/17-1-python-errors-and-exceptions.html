Python Exception Handling

In Python, an exception is an event that occurs during the execution of a program that disrupts the normal flow of the program's instructions.

What are Exceptions?

When an error occurs in Python, it raises an Exception.

If not handled, the program stops. Python provides mechanisms to "catch" and "handle" exceptions using try, except, else, and finally blocks.

Important Note

SyntaxError happens before the code runs (during compilation/interpretation).

try/except only works on errors that happen at runtime (when the code is already being executed).

1. try and except

try: Block of code that may cause an error.

except: Block of code that runs if an error occurs.

# Syntax
try:
    # block of code that error might occur
except exception_type:
    # block of code to run if the error occurs 
# Example
try:
    num = int("abc")
except ValueError:
    print("Oops! Not a number.") 
 Output
Oops! Not a number.

2. else

Runs only if no error occurs inside try

try:
    num = int("123")
except ValueError:
    print("Error: not a number")
else:
    print("Conversion successful:", num) 
 Output
Conversion successful: 123

3. finally

Always runs, whether an error occurs or not

Useful for cleanup (closing files, releasing resources).

try:
    num = 10 / 0
except ZeroDivisionError:
    print("Cannot divide by zero.")
finally:
    print("This block always runs.") 
 Output
Cannot divide by zero.
This block always runs.

4. raise

Used to raise an exception manually.

age = -5
if age < 0:
    raise ValueError("Age cannot be negative!") 
 Output
ValueError: Age cannot be negative!

Using as e with Exceptions

Capture the error object for more details

try:
    num = int("abc")   # This will cause ValueError
except ValueError as e:
    print("Error occurred:", e) 
 Output
Error occurred: invalid literal for int() with base 10: 'abc'

Using except Exception as e

Catch any error (since all exceptions inherit from Exception)

Useful when we are not sure which exception will be raised.

try:
    result = 10 / 0    # This will cause ZeroDivisionError
    num = int("abc")   # This would cause ValueError (but code never reaches here)
except Exception as e:
    print("Error occurred:", e) 
 Output
Error occurred: division by zero

5. Creating Our Own Exception

Create custom exceptions by inheriting from Exception

Useful when you want to define application-specific errors.

# Define custom exception
class NegativeAgeError(Exception):
    pass

try:
    age = -5
    if age < 0:
        raise NegativeAgeError("Age cannot be negative!")
except NegativeAgeError as e:
    print("Custom Exception Caught:", e) 
 Output
Custom Exception Caught: Age cannot be negative!

Common Python Exceptions

Exception When it Occurs Example
SyntaxError Code is not written correctly (invalid syntax) print "hi"
IndentationError Wrong indentation (spaces/tabs) if True:
print("Hi")
NameError Variable or name is not defined print(x) (x not defined)
TypeError Wrong data type used in an operation "10" + 5
ValueError Correct type but wrong value int("abc")
ZeroDivisionError Division by zero 10 / 0
IndexError Index not found in a list/tuple arr=[1,2]; print(arr[5])
KeyError Dictionary key not found d={"a":1}; print(d["b"])
AttributeError Attribute/method does not exist on an object "hi".append("!")
ImportError Problem importing a module import not_a_real_module
ModuleNotFoundError Specific case of ImportError → module not found import somemodule
FileNotFoundError File does not exist open("nofile.txt")
RuntimeError General error not covered by other exceptions raise RuntimeError("error")

# 1. Division with zero handling
def safe_divide(a, b):
    try:
        result = a / b
        return f"Result: {result}"
    except ZeroDivisionError:
        return "Error: Cannot divide by zero!"
    except TypeError:
        return "Error: Please provide numbers only"

# 2. Integer conversion with error handling
def get_integer_input(prompt):
    try:
        user_input = input(prompt)
        number = int(user_input)
        return f"Success! You entered: {number}"
    except ValueError:
        return f"Error: '{user_input}' is not a valid integer"

# 3. Safe list access
def safe_list_access(lst, index):
    try:
        return f"Element at index {index}: {lst[index]}"
    except IndexError:
        return f"Error: Index {index} is out of range for list of length {len(lst)}"
    except TypeError:
        return "Error: Please provide valid list and integer index"

# 4. Custom exception for negative numbers
class NegativeNumberError(Exception):
    def __init__(self, value):
        self.value = value
        super().__init__(f"Negative numbers not allowed: {value}")

def check_positive(number):
    if number < 0:
        raise NegativeNumberError(number)
    return f"Valid positive number: {number}"

# 5. File reading with error handling
def read_file_safely(filename):
    try:
        with open(filename, 'r') as file:
            content = file.read()
            return f"File content:\n{content}"
    except FileNotFoundError:
        return f"Error: File '{filename}' not found"
    except PermissionError:
        return f"Error: No permission to read '{filename}'"
    except Exception as e:
        return f"Unexpected error: {e}"

Key Points
  • try: Code that might cause an error
  • except: Handle specific exceptions
  • else: Run if no exception occurs
  • finally: Always runs (cleanup code)
  • raise: Manually raise exceptions
  • as e: Get error details
  • Create custom exceptions by inheriting from Exception
  • Use except Exception as e to catch any error
