1. Encapsulation

Definition

Encapsulation refers to the bundling of attributes and methods inside a single class.

It prevents outer classes from accessing and changing attributes and methods of a class. This also helps to achieve data hiding.

Private Attributes in Python
class BankAccount:
    def __init__(self, name, balance=0):
        self.name = name
        self.__balance = balance  # Private variable (double underscore)
    
    def get_balance(self):
        return self.__balance
    
    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            return f"Deposited {amount}. New balance: {self.__balance}"
        return "Invalid amount"
    
    def withdraw(self, amount):
        if 0 < amount <= self.__balance:
            self.__balance -= amount
            return f"Withdrew {amount}. New balance: {self.__balance}"
        return "Not enough balance"

# Example usage
account = BankAccount("John", 1000)
print(account.deposit(500))    # Deposited 500. New balance: 1500
print(account.withdraw(200))   # Withdrew 200. New balance: 1300
print(account.get_balance())   # 1300
# print(account.__balance)     # This would cause AttributeError! 
 Output
Deposited 500. New balance: 1500
Withdrew 200. New balance: 1300
1300
Access Control in Python
  • No modifier: Public - accessible from anywhere
  • Single underscore (_variable): Protected - internal use (convention)
  • Double underscore (__variable): Private - name mangling makes it harder to access

2. Inheritance

Definition

Inheritance is a way of creating a new class for using details of an existing class without modifying it.

Creating new classes from existing ones.

Types of Inheritance
Single Inheritance

One parent → One child

Multiple Inheritance

Multiple parents → One child

Multilevel Inheritance

Grandparent → Parent → Child

Hierarchical Inheritance

One parent → Multiple children

Hybrid Inheritance

Combination of multiple types

a. Single Inheritance

When a child class inherits from a single parent class, it can access the parent’s methods and attributes.

class Parent:
    def greet(self):
        print("Hello from Parent")

class Child(Parent):  # Child inherits from Parent
    def display(self):
        print("This is Child class")

# Usage
c = Child()
c.greet()    # Accessing Parent method
c.display()  # Accessing Child method 
 Output
Hello from Parent
This is Child class
b. Multiple Inheritance

When a child class inherits from more than one parent class. This allows the child to use methods from multiple parents.

class Father:
    def drive(self):
        print("Father can drive")

class Mother:
    def cook(self):
        print("Mother can cook")

class Child(Father, Mother):  # Inherits from both
    def play(self):
        print("Child loves to play")

# Usage
c = Child()
c.drive()  # From Father
c.cook()   # From Mother
c.play()   # From Child 
 Output
Father can drive
Mother can cook
Child loves to play
c. Multilevel Inheritance

When a class is derived from another derived class (like a chain).

class GrandParent:
    def wisdom(self):
        print("GrandParent shares wisdom")

class Parent(GrandParent):  # Parent inherits from GrandParent
    def guidance(self):
        print("Parent gives guidance")

class Child(Parent):  # Child inherits from Parent
    def learn(self):
        print("Child learns")

# Usage
c = Child()
c.wisdom()   # From GrandParent
c.guidance() # From Parent
c.learn()    # From Child 
 Output
GrandParent shares wisdom
Parent gives guidance
Child learns
d. Hierarchical Inheritance

When multiple child classes inherit from the same parent.

class Parent:
    def greet(self):
        print("Hello from Parent")

class Child1(Parent):  # First child
    def feature1(self):
        print("Feature from Child1")

class Child2(Parent):  # Second child
    def feature2(self):
        print("Feature from Child2")

# Usage
c1 = Child1()
c2 = Child2()
c1.greet()    # From Parent
c1.feature1() # From Child1
c2.greet()    # From Parent
c2.feature2() # From Child2 
 Output
Hello from Parent
Feature from Child1
Hello from Parent
Feature from Child2
e. Hybrid Inheritance

A combination of multiple inheritance types (like multiple + multilevel).

It forms a complex structure where a class inherits from more than one path.

class A:
    def showA(self):
        print("Class A")

class B(A):  # B inherits from A
    def showB(self):
        print("Class B")

class C(A):  # C inherits from A
    def showC(self):
        print("Class C")

class D(B, C):  # D inherits from both B and C
    def showD(self):
        print("Class D")

# Usage
d = D()
d.showA()  # From A (through B or C)
d.showB()  # From B
d.showC()  # From C
d.showD()  # From D 
 Output
Class A
Class B
Class C
Class D
super() Method

In Python, super() is a built-in function that provides a way to access methods and properties of a parent or superclass from within a child or subclass.

class Parent:
    def __init__(self, name):
        self.name = name
        print(f"Parent initialized with name: {self.name}")

class Child(Parent):
    def __init__(self, name, age):
        super().__init__(name)  # Call Parent's __init__
        self.age = age
        print(f"Child initialized with age: {self.age}")

# Usage
c = Child("John", 12) 
 Output
Parent initialized with name: John
Child initialized with age: 12
super() with Multiple Inheritance
class Father:
    def __init__(self):
        print("Father initialized")

class Mother:
    def __init__(self):
        print("Mother initialized")

class Child(Father, Mother):
    def __init__(self):
        super().__init__()  # Calls Father first (based on MRO)
        print("Child initialized")

# Usage
c = Child() 
 Output
Father initialized
Child initialized
Method Resolution Order (MRO)

Python follows a specific order when looking for methods in inheritance chains. Use ClassName.__mro__ to see the order.

print(Child.__mro__)
# Shows the method resolution order

3. Polymorphism

Definition

Polymorphism means "many forms". In Python, it allows the same function/method name to perform different tasks depending on the object or class using it.

Built-in Polymorphism Example
# len() function works with different objects
x = "Hello World!"
print(len(x))  # String length

x = [1, 2, 4, 5, 6]
print(len(x))  # List length

x = (1, 2, 4, 5, 6)
print(len(x))  # Tuple length 
 Output
12
5
5
Polymorphism with Functions
def add(a, b):
    return a + b

# Works with integers
print(add(5, 10))           # 15

# Works with strings
print(add("Hello ", "World"))  # Hello World

# Works with lists
print(add([1, 2], [3, 4]))    # [1, 2, 3, 4] 
 Output
15
Hello World
[1, 2, 3, 4]
Method Overloading vs Method Overriding
Comparison
Feature Method Overloading Method Overriding
Definition Same method name, different parameters Same method name in parent & child classes
When resolved Compile-time Polymorphism Runtime Polymorphism
Arguments Different number/type of arguments Must have the same arguments
Where used? Inside the same class Between parent & child classes
Python Support Not directly supported (simulate with default args / *args) Fully supported
Method Overloading (Simulated)

Meaning: Same method name, but different number or type of parameters.

In Python: True overloading doesn’t exist (like in Java or C++).

Achieved during compilation.

Instead, Python handles this using default arguments or *args.

class Math:
    def add(self, a, b=0, c=0):  # Same method, flexible arguments
        return a + b + c

m = Math()
print(m.add(5))        # 5 (a=5, b=0, c=0)
print(m.add(5, 10))    # 15 (a=5, b=10, c=0)
print(m.add(5, 10, 15)) # 30 (a=5, b=10, c=15) 
 Output
5
15
30
Method Overriding

Meaning: Child class provides a new implementation for a method that already exists in the parent class.

Achieved while the program runs.

This allows customizing or changing behavior in subclasses.

class Vehicle:
    def start(self):
        return "Starting vehicle..."

class Car(Vehicle):
    def start(self):  # Overriding parent method
        return "Car engine started"

class Bike(Vehicle):
    def start(self):  # Overriding parent method
        return "Bike engine started"

# Usage
vehicle = Vehicle()
car = Car()
bike = Bike()

print(vehicle.start())  # Starting vehicle...
print(car.start())      # Car engine started
print(bike.start())     # Bike engine started 
 Output
Starting vehicle...
Car engine started
Bike engine started

4. Abstraction

Definition

Abstraction means hiding the complex implementation details and showing only the essential features of the object.

In Python, we achieve abstraction using abstract classes and interfaces.

Abstract Base Classes (ABC)
from abc import ABC, abstractmethod

class Shape(ABC):  # Abstract class
    @abstractmethod
    def area(self):
        pass  # No implementation - must be overridden
    
    @abstractmethod
    def perimeter(self):
        pass
    
    def description(self):  # Concrete method
        return "This is a shape"

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):  # Must implement abstract method
        return 3.14 * self.radius * self.radius
    
    def perimeter(self):  # Must implement abstract method
        return 2 * 3.14 * self.radius

class Rectangle(Shape):
    def __init__(self, length, width):
        self.length = length
        self.width = width
    
    def area(self):
        return self.length * self.width
    
    def perimeter(self):
        return 2 * (self.length + self.width)

# Usage
circle = Circle(5)
rectangle = Rectangle(4, 6)

print(f"Circle area: {circle.area()}")
print(f"Rectangle area: {rectangle.area()}")
print(f"Circle description: {circle.description()}")

# shape = Shape()  # This would cause TypeError - cannot instantiate abstract class 
 Output
Circle area: 78.5
Rectangle area: 24
Circle description: This is a shape
Key Points about Abstraction
  • Abstract classes cannot be instantiated
  • Abstract methods must be implemented by child classes
  • Provides a template for other classes
  • Hides complex implementation details

Try It Yourself

Create a complete OOP system demonstrating all four pillars:

  1. Encapsulation: Create a BankAccount class with private balance
  2. Inheritance: Create SavingsAccount and CurrentAccount that inherit from BankAccount
  3. Polymorphism: Override withdraw() method in child classes with different rules
  4. Abstraction: Create an abstract Account class that defines required methods
from abc import ABC, abstractmethod

# 4. Abstraction - Abstract base class
class Account(ABC):
    @abstractmethod
    def withdraw(self, amount):
        pass
    
    @abstractmethod
    def deposit(self, amount):
        pass
    
    def get_account_type(self):
        return "Generic Account"

# 1. Encapsulation & 2. Inheritance
class BankAccount(Account):
    def __init__(self, account_holder, balance=0):
        self.account_holder = account_holder
        self.__balance = balance  # Private variable
    
    def get_balance(self):
        return self.__balance
    
    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            return f"Deposited ${amount}. New balance: ${self.__balance}"
        return "Invalid amount"
    
    def withdraw(self, amount):
        if 0 < amount <= self.__balance:
            self.__balance -= amount
            return f"Withdrew ${amount}. New balance: ${self.__balance}"
        return "Insufficient funds"
    
    def get_account_type(self):
        return "Basic Bank Account"

# 2. Inheritance & 3. Polymorphism
class SavingsAccount(BankAccount):
    def __init__(self, account_holder, balance=0, interest_rate=0.02):
        super().__init__(account_holder, balance)
        self.interest_rate = interest_rate
    
    def withdraw(self, amount):  # 3. Polymorphism - Overriding
        if self.get_balance() - amount < 100:  # Minimum balance rule
            return "Cannot withdraw: Minimum balance of $100 required"
        return super().withdraw(amount)
    
    def add_interest(self):
        interest = self.get_balance() * self.interest_rate
        self.deposit(interest)
        return f"Added interest: ${interest}"
    
    def get_account_type(self):
        return "Savings Account"

class CurrentAccount(BankAccount):
    def __init__(self, account_holder, balance=0, overdraft_limit=500):
        super().__init__(account_holder, balance)
        self.overdraft_limit = overdraft_limit
    
    def withdraw(self, amount):  # 3. Polymorphism - Overriding
        available_balance = self.get_balance() + self.overdraft_limit
        if 0 < amount <= available_balance:
            # For current account, allow overdraft
            new_balance = self.get_balance() - amount
            return f"Withdrew ${amount}. New balance: ${new_balance}"
        return "Exceeds overdraft limit"
    
    def get_account_type(self):
        return "Current Account"

# Testing the system
print("=== BANKING SYSTEM DEMO ===")

savings = SavingsAccount("Alice", 500)
current = CurrentAccount("Bob", 300, 200)

print(f"\n{savings.get_account_type()}:")
print(savings.deposit(200))
print(savings.withdraw(400))  # Should fail due to min balance
print(savings.withdraw(300))  # Should work
print(savings.add_interest())

print(f"\n{current.get_account_type()}:")
print(current.deposit(100))
print(current.withdraw(500))  # Should work (uses overdraft)
print(current.withdraw(200))  # Should fail (exceeds limit) 
 Output
=== BANKING SYSTEM DEMO ===

Savings Account:
Deposited $200. New balance: $700
Cannot withdraw: Minimum balance of $100 required
Withdrew $300. New balance: $400
Added interest: $8.0

Current Account:
Deposited $100. New balance: $400
Withdrew $500. New balance: $-100
Exceeds overdraft limit

Key Takeaways
  • Encapsulation: Bundle data and methods, control access with private variables
  • Inheritance: Create hierarchies, reuse code through parent-child relationships
  • Polymorphism: Same interface, different implementations through overloading and overriding
  • Abstraction: Hide complexity, define templates with abstract classes
  • These four pillars work together to create robust, maintainable, and scalable code
  • Python supports all OOP concepts with its unique syntax and features
  • Understanding these concepts is crucial for professional Python development
