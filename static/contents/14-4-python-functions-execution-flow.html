Call Stack Visualization - Step by Step

Python uses a call stack (also called execution stack) to keep track of function calls.

Think of it like a pile of plates — the most recent function call is placed on top of the stack, and when it finishes, it is removed from the top.

Call Stack :A stack is a data structure that follows LIFO (Last In, First Out).

def function1():
    print("Start function1")
    function2()
    print("End function1")

def function2():
    print("Start function2")
    function3()
    print("End function2")

def function3():
    print("Start function3")
    print("End function3")

print("Main program starts")
function1()
print("Main program ends")
Step 1: Main Program Starts
What happens:
  • Main program begins execution
  • Call stack is initialized
Output:
Main program starts
Call Stack:
main

↑ Top of stack

Step 2: function1() is Called
What happens:
  • function1() is added to call stack
  • function1() starts executing
  • Prints "Start function1"
Output:
Start function1
Call Stack:
function1()
main

↑ Top of stack

Step 3: function2() is Called from function1()
What happens:
  • function2() is added to call stack
  • function2() starts executing
  • Prints "Start function2"
  • function1() is paused
Output:
Start function2
Call Stack:
function2()
function1()
main

↑ Top of stack

Step 4: function3() is Called from function2()
What happens:
  • function3() is added to call stack
  • function3() starts executing
  • Prints "Start function3"
  • function2() is paused
Output:
Start function3
Call Stack:
function3()
function2()
function1()
main

↑ Top of stack (MAX DEPTH)

Step 5: function3() Completes
What happens:
  • function3() prints "End function3"
  • function3() is removed from stack
  • Control returns to function2()
Output:
End function3
Call Stack:
function2()
function1()

Step 6: function2() Completes
What happens:
  • function2() prints "End function2"
  • function2() is removed from stack
  • Control returns to function1()
Output:
End function2

main

↑ Top of stack

Step 7: function1() Completes
What happens:
  • function1() prints "End function1"
  • function1() is removed from stack

Call Stack:
main

↑ Top of stack

Step 8: Main Program Ends
What happens:
  • Main program prints final message
  • Main program is removed from stack
  • Call stack becomes empty
  • Program execution completes

Key Concepts
Call Stack Rules:
  • LIFO: Last In, First Out
  • Top executes: Only top function runs
  • Pause/resume: Functions pause when calling others
  • Automatic management: Python handles stack automatically
Visual Cues:
  • main - Main program
  • function1 - First level function
  • function2 - Second level function
  • function3 - Third level function

Try It Yourself

Draw the call stack for each step of this program:

def math_operations():
    print("Starting math operations")
    result = add_numbers(10, 5)
    print(f"Result: {result}")
    print("Math operations complete")

def add_numbers(a, b):
    print(f"Adding {a} + {b}")
    square = calculate_square(a)
    return square + b

def calculate_square(x):
    print(f"Calculating square of {x}")
    return x * x

print("Program begins")
math_operations()
print("Program ends")
# Expected Output:
Program begins
Starting math operations
Adding 10 + 5
Calculating square of 10
Result: 105
Math operations complete
Program ends

# 6. [main → math_operations]
