Python Object-Oriented Programming (OOP)

Learn how to organize code using classes and objects for better structure and reusability.

What is Object-Oriented Programming?

Object-Oriented Programming (OOP) is a way of writing code that organizes everything into objects and classes.

Think of classes as blueprints and objects as actual houses built from those blueprints.

Advantages of OOP

Clear Structure

Provides a clear structure to programs

Easy Maintenance

Makes code easier to maintain, reuse, and debug

DRY Principle

Helps keep your code DRY (Don't Repeat Yourself)

Reusable Code

Allows you to build reusable applications with less code

DRY Principle

The DRY principle means you should avoid writing the same code more than once. Move repeated code into functions or classes and reuse it.

Classes in Python

What is a Class?

A class is a blueprint that defines what an object will be like and what it can do.

Class members are variables and methods that belong to a class.

Basic Class Syntax
class MyClass:
    x = 5  # class attribute
    
    def my_method(self):
        print("Hello from method!")
Example: Human Class
class Human:
    gender = "Male"      # class attribute
    age = 22             # class attribute
    hasHeart = True      # class attribute
    hasBrain = True      # class attribute
    
    def speak(self):
        print("Hello, I am Human")
    
    def walk(self):
        print("I am walking")
    
    def eat(self):
        print("I am eating")
Terminology
  • Functions inside classes are called methods
  • Variables inside classes are called attributes
  • self refers to the current object instance

Objects in Python

What is an Object?

An object is a real instance created from a class blueprint that has actual data and can perform real actions.

1. Creating Objects
class Human:
    gender = "Male"
    age = 22
    hasHeart = True
    hasBrain = True
    
    def speak(self):
        print("Hello, I am Human")
    
    def walk(self):
        print("I am walking")
    
    def eat(self):
        print("I am eating")

# Create an object
human = Human()

human = Human()

Improved Version: Using __init__()

Key Concepts
  • self refers to the specific object (holds the reference of an object)
  • __init__ is a special method that automatically runs when you create a new object
  • It's called the constructor or initializer
Improved Human Class
class Human:
    # Class variables (shared by ALL humans)
    hasHeart = True
    hasBrain = True
    
    def __init__(self, name, gender, age):  # Constructor
        # Instance variables (unique to EACH human)
        self.name = name
        self.gender = gender
        self.age = age
    
    def speak(self):
        print(f"Hello, I am {self.name}")
    
    def walk(self):
        print(f"{self.name} is walking")
    
    def eat(self):
        print(f"{self.name} is eating")
    
    def introduce(self):
        print(f"My name is {self.name}, I am {self.gender} with age {self.age}")
Creating Different Humans
# Human 1
person1 = Human("Ananya", "Female", 25)

Example 2: Mobile Phone Class

class MobilePhone:
    # Class variables (same for all phones)
    can_call = True
    can_text = True
    has_screen = True
    
    def __init__(self, brand, model, storage, color):
        # Instance variables (different for each phone)
        self.brand = brand
        self.model = model
        self.storage = storage
        self.color = color
        self.is_on = False
    
    def turn_on(self):
        self.is_on = True
        return f"{self.brand} {self.model} is now ON"
    
    def describe(self):
        return f"{self.color} {self.brand} {self.model} with {self.storage}GB"

# Creating different phones from same blueprint
phone1 = MobilePhone("Apple", "iPhone 14", 128, "Black")
phone2 = MobilePhone("Samsung", "Galaxy S23", 256, "White")
phone3 = MobilePhone("Google", "Pixel 7", 64, "Blue")

print("=== PHONE DESCRIPTIONS ===")
print(phone1.describe())
print(phone2.describe())
print(phone3.describe())

print("\n=== TURNING THEM ON ===")
print(phone1.turn_on())
print(phone2.turn_on())
print(phone3.turn_on())

print(f"\nAll phones can make calls: {MobilePhone.can_call}") 
 Output
=== PHONE DESCRIPTIONS ===

White Samsung Galaxy S23 with 256GB

=== TURNING THEM ON ===
Apple iPhone 14 is now ON
Samsung Galaxy S23 is now ON
Google Pixel 7 is now ON

All phones can make calls: True

Class Variables vs Instance Variables

Understanding the Difference
Type Definition Access Example
Class Variables Shared by ALL objects ClassName.variable Human.hasHeart
Instance Variables Unique to EACH object object.variable person1.name
Demonstration
class Car:
    # Class variable (shared)
    wheels = 4
    
    def __init__(self, brand, color):
        # Instance variables (unique)
        self.brand = brand
        self.color = color

# Create cars
car1 = Car("Toyota", "Red")
car2 = Car("Honda", "Blue")

print("=== INSTANCE VARIABLES (Different) ===")
print(f"Car1: {car1.brand}, {car1.color}")
print(f"Car2: {car2.brand}, {car2.color}")

print("\n=== CLASS VARIABLES (Same) ===")
print(f"Car1 wheels: {car1.wheels}")
print(f"Car2 wheels: {car2.wheels}")
print(f"All cars have: {Car.wheels} wheels")

# Change class variable (affects ALL cars)
Car.wheels = 6
print(f"\nAfter change - All cars now have: {Car.wheels} wheels") 
 Output
=== INSTANCE VARIABLES (Different) ===
Car1: Toyota, Red
Car2: Honda, Blue

=== CLASS VARIABLES (Same) ===
Car1 wheels: 4
Car2 wheels: 4
All cars have: 4 wheels

After change - All cars now have: 6 wheels

The Universal Pattern

class Something:
    # SHARED stuff (class variables)
    shared_thing = True
    
    def __init__(self, unique1, unique2, unique3):
        # UNIQUE stuff (instance variables)
        self.unique1 = unique1
        self.unique2 = unique2
        self.unique3 = unique3
    
    def do_something(self):
        return f"Working with {self.unique1}"

# Create different objects
obj1 = Something("A", "B", "C")
obj2 = Something("X", "Y", "Z")
obj3 = Something("1", "2", "3")

print(obj1.do_something())  # Working with A
print(obj2.do_something())  # Working with X
print(obj3.do_something())  # Working with 1 
 Output
Working with A
Working with X
Working with 1

Try It Yourself

Create a BankAccount class with the following:

  1. Class variable: bank_name = "Python Bank" (shared by all accounts)
  2. Instance variables in constructor: account_holder, balance
  3. Methods:
    • deposit(amount) - add to balance
    • withdraw(amount) - subtract from balance (check if sufficient funds)
    • get_balance() - return current balance
    • display_info() - show account holder and balance

Create multiple bank accounts and test all methods.

class BankAccount:
    # Class variable (shared by all accounts)
    bank_name = "Python Bank"
    
    def __init__(self, account_holder, balance=0):
        # Instance variables (unique to each account)
        self.account_holder = account_holder
        self.balance = balance
    
    def deposit(self, amount):
        self.balance += amount
        return f"Deposited ${amount}. New balance: ${self.balance}"
    
    def withdraw(self, amount):
        if amount <= self.balance:
            self.balance -= amount
            return f"Withdrew ${amount}. New balance: ${self.balance}"
        else:
            return "Insufficient funds!"
    
    def get_balance(self):
        return self.balance
    
    def display_info(self):
        return f"Bank: {self.bank_name}, Holder: {self.account_holder}, Balance: ${self.balance}"

# Create accounts
account1 = BankAccount("Alice", 1000)
account2 = BankAccount("Bob", 500)

print("=== ACCOUNT INFORMATION ===")
print(account1.display_info())
print(account2.display_info())

print("\n=== TRANSACTIONS ===")
print(account1.deposit(200))
print(account1.withdraw(300))
print(account2.withdraw(600))  # Should show insufficient funds

print("\n=== FINAL BALANCES ===")
print(f"Alice's balance: ${account1.get_balance()}")
print(f"Bob's balance: ${account2.get_balance()}")

print(f"\nAll accounts belong to: {BankAccount.bank_name}") 
 Output
=== ACCOUNT INFORMATION ===
Bank: Python Bank, Holder: Alice, Balance: $1000
Bank: Python Bank, Holder: Bob, Balance: $500

=== TRANSACTIONS ===
Deposited $200. New balance: $1200
Withdrew $300. New balance: $900
Insufficient funds!

=== FINAL BALANCES ===
Alice's balance: $900
Bob's balance: $500

All accounts belong to: Python Bank

Key Takeaways
  • Class = Blueprint, Object = Instance
  • __init__ method initializes object attributes
  • self refers to the current object instance
  • Class variables are shared, instance variables are unique
  • OOP provides clear structure and promotes code reuse
  • Follows DRY principle - Don't Repeat Yourself
  • One class can create many different objects
  • Methods define actions, attributes define data
