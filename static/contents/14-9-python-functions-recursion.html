Python Recursion: Function Calling Itself

Recursion is a process where a function calls itself to solve a problem.

The 3 Golden Rules of Recursion

Rule 1

Function calls itself

Rule 2

Has a base case (stopping condition)

Rule 3

Moves toward base case with each call

Basic Syntax
def my_function(params):
    # base condition (when to stop)
    if base_case_condition:
        return result
    
    # recursive call (function calls itself)
    # with parameter moving toward base case
    return my_function(modified_params)

Example 1: Countdown

def countdown(n):
    # Rule 2: BASE CASE - when to stop
    if n == 0:
        print("Done!")
        return
    
    # Do some work
    print(n)
    
    # Rule 1: CALL ITSELF
    # Rule 3: MOVE TOWARD BASE CASE (n-1 gets closer to 0)
    countdown(n - 1)

print("Countdown from 3:")
countdown(3) 
 Output
Countdown from 3:
3
2
1
Done!
How It Works - Step by Step
  1. countdown(3) prints 3, calls countdown(2)
  2. countdown(2) prints 2, calls countdown(1)
  3. countdown(1) prints 1, calls countdown(0)
  4. countdown(0) prints "Done!" and returns
  5. All functions complete and program ends

Example 2: Print Numbers 1 to N

def print_numbers(n):
    # base condition
    if n == 0:
        return
    
    print_numbers(n - 1)  # recursive call first
    print(n)              # then print

print("Numbers 1 to 5:")
print_numbers(5) 
 Output
Numbers 1 to 5:
1
2
3
4
5
Why Does It Print in Order?

The recursive call happens before printing, so:

  • print_numbers(5) calls print_numbers(4) before printing 5
  • print_numbers(4) calls print_numbers(3) before printing 4
  • This continues until base case, then printing happens on the way back
Compare: Print in Reverse Order
def print_numbers_reverse(n):
    # base condition
    if n == 0:
        return
    
    print(n)              # print first
    print_numbers_reverse(n - 1)  # then recursive call

print("Numbers 5 to 1:")
print_numbers_reverse(5) 
 Output
Numbers 5 to 1:
5
4
3
2
1

More Practical Examples

Example 3: Factorial
def factorial(n):
    # base case: 0! = 1 and 1! = 1
    if n == 0 or n == 1:
        return 1
    
    # recursive case: n! = n * (n-1)!
    return n * factorial(n - 1)

print(f"5! = {factorial(5)}")
print(f"3! = {factorial(3)}")
print(f"0! = {factorial(0)}") 
 Output
5! = 120
3! = 6
0! = 1
Example 4: Sum of Numbers
def sum_numbers(n):
    # base case
    if n == 0:
        return 0
    
    # recursive case: sum(n) = n + sum(n-1)
    return n + sum_numbers(n - 1)

print(f"Sum of 1 to 5: {sum_numbers(5)}")
print(f"Sum of 1 to 10: {sum_numbers(10)}") 
 Output
Sum of 1 to 5: 15
Sum of 1 to 10: 55
Example 5: Fibonacci Sequence
def fibonacci(n):
    # base cases
    if n == 0:
        return 0
    elif n == 1:
        return 1
    
    # recursive case: fib(n) = fib(n-1) + fib(n-2)
    return fibonacci(n - 1) + fibonacci(n - 2)

print("Fibonacci sequence:")
for i in range(10):
    print(f"fib({i}) = {fibonacci(i)}") 
 Output
Fibonacci sequence:
fib(0) = 0
fib(1) = 1
fib(2) = 1
fib(3) = 2
fib(4) = 3
fib(5) = 5
fib(6) = 8
fib(7) = 13
fib(8) = 21
fib(9) = 34

Common Recursion Mistakes

Mistake 1: No Base Case (Infinite Recursion)
def infinite_recursion(n):
    print(n)
    infinite_recursion(n + 1)  # No base case!

# This would run forever until RecursionError
# infinite_recursion(1)
Mistake 2: Not Moving Toward Base Case
def bad_recursion(n):
    if n <= 0:  # Has base case
        return
    print(n)
    bad_recursion(n)  # But doesn't move toward base case!

# This would also cause RecursionError
# bad_recursion(5)
RecursionError

Python has a recursion limit (usually 1000 calls). If you exceed this, you'll get:

RecursionError: maximum recursion depth exceeded

Quick Reference
Problem Base Case Recursive Case
Countdown n == 0 countdown(n-1)
Factorial n == 0 or n == 1 n * factorial(n-1)
Sum of Numbers n == 0 n + sum_numbers(n-1)
Fibonacci n == 0 or n == 1 fib(n-1) + fib(n-2)

Try It Yourself

Create recursive functions to solve these problems:

  1. Calculate the product of numbers from 1 to n (like factorial but for any range)
  2. Count how many digits are in a number (without converting to string)
  3. Reverse a string using recursion
  4. Check if a string is a palindrome (reads same forwards and backwards)
  5. Calculate the power of a number: power(x, n) = xⁿ
# 1. Product of numbers from 1 to n
def product(n):
    if n == 1:
        return 1
    return n * product(n - 1)

print(f"Product 1 to 5: {product(5)}")

# 2. Count digits in a number
def count_digits(n):
    if n < 10:
        return 1
    return 1 + count_digits(n // 10)

print(f"Digits in 12345: {count_digits(12345)}")

# 3. Reverse a string
def reverse_string(s):
    if len(s) <= 1:
        return s
    return reverse_string(s[1:]) + s[0]

print(f"Reverse 'hello': {reverse_string('hello')}")

# 4. Check if palindrome
def is_palindrome(s):
    if len(s) <= 1:
        return True
    if s[0] != s[-1]:
        return False
    return is_palindrome(s[1:-1])

print(f"'radar' is palindrome: {is_palindrome('radar')}")
print(f"'python' is palindrome: {is_palindrome('python')}")

# 5. Power calculation
def power(x, n):
    if n == 0:
        return 1
    return x * power(x, n - 1)

print(f"2^5 = {power(2, 5)}")
print(f"3^3 = {power(3, 3)}") 
 Output
Product 1 to 5: 120
Digits in 12345: 5
Reverse 'hello': olleh
'radar' is palindrome: True
'python' is palindrome: False
2^5 = 32
3^3 = 27

Key Takeaways
  • Recursion is when a function calls itself
  • Three Golden Rules: Call itself, Base case, Move toward base case
  • Base case stops the recursion (like n == 0)
  • Recursive case breaks problem into smaller versions
  • Common uses: factorial, Fibonacci, tree traversal, sorting algorithms
  • Watch out for: Infinite recursion, Stack overflow
  • Recursion can often be replaced with loops (iteration)
  • Some problems are naturally recursive and easier to solve with recursion
